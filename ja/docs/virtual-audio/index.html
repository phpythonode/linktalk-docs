仮想オーディオ設定

仮想オーディオデバイスはソフトウェア間の音声ブリッジで、「Input で音声を受け取り → Output で音声を送り出す」というルールに従います。語橋はデュアルチャンネル独立設計を採用しており、2 つの方向は完全に独立して干渉しません：

  • チャンネル A（私 → 相手）：実際のマイク → 認識・翻訳 → 翻訳出力（会議/配信に送信）
  • チャンネル B（相手 → 私）：相手の音声入力 → 認識・翻訳 → 相手の翻訳出力（ローカル再生）

:::tip 基本原則 私の翻訳は 1 つの仮想オーディオデバイスを通り、相手の翻訳は別のデバイスを通ります。完全に独立、絶対に交差しない。 :::

仮想オーディオデバイスはいくつ必要ですか？​

シナリオ必要数推奨
簡単なテスト（一方向）1 個「私 → 相手」または「相手 → 私」のみテスト
一方向ライブ/会議1 個私の翻訳音声を視聴者/会議に送信
双方向国際会議2 個両方向を完全に独立させ、クロストークなし
複雑なマルチストリームシナリオ3 個以上各方向に独立した仮想デバイスを割り当て

ツールのダウンロード​

Windows：VB-Audio Cable（無料、推奨）​

Cable A と Cable B の 2 つをダウンロード。インストール後、システムが自動的に 2 つの独立した仮想オーディオデバイスとして認識します。軽量で広告なし。

  • 公式サイト：https://vb-audio.com/Cable/
  • オプション：VoiceMeeter Banana（視覚的なデバッグ、無音/クロストークの診断に便利）

macOS：BlackHole（無料、推奨）​

BlackHole は Mac で最も人気の仮想オーディオデバイスで、Apple Silicon（M1/M2/M3/M4）をネイティブサポートします。

  • 公式サイト：https://existential.audio/blackhole/
  • 推奨：BlackHole 2ch
  • Homebrew でもインストール可能： 
    brew install blackhole-2ch

:::tip Mac デュアルデバイス設定 双方向会議で 2 つの独立した仮想デバイスが必要な場合、BlackHole 2ch + BlackHole 16ch をインストールして 2 つのチャンネルに対応させるか、「Audio MIDI 設定」で集約デバイスを作成します。 :::

シナリオ 1：双方向国際会議（完全双方向翻訳）​

目標：私が話す → 翻訳音声が相手に届く；相手が話す → 翻訳音声が私に届く。両方向完全独立。

必要なもの：仮想デバイス A（私の翻訳 → 会議）+ 仮想デバイス B（相手の音声 → 通訳）+ 実際のマイク + ヘッドフォン

チャンネル A：私 → 相手

実際のマイク → 語橋「マイク入力」→ 認識・翻訳
→ 語橋「翻訳出力」→ 仮想デバイス A Input
→ 仮想デバイス A Output → 会議ソフト「マイク入力」→ 相手が翻訳を聞く

チャンネル B：相手 → 私

相手が話す → 会議ソフト「出力デバイス」→ 仮想デバイス B Input
→ 仮想デバイス B Output → 語橋「相手の音声入力」→ 認識・翻訳
→ 語橋「相手の翻訳出力」→ 実際のヘッドフォン → 私が翻訳を聞く

設定対照表

ソフトウェア設定項目選択デバイス
語橋マイク入力実際のマイク
語橋翻訳出力仮想デバイス A Input
語橋私の翻訳再生オフ（エコー防止）
語橋相手の音声入力仮想デバイス B Output
語橋相手の翻訳出力実際のヘッドフォン
会議ソフトマイク入力仮想デバイス A Output
会議ソフト出力デバイス仮想デバイス B Input

シナリオ 2：一方向ライブ/会議（私 → 相手）​

プレゼンテーションやライブコマースに適しています。私の翻訳音声を送り出すだけで、相手の音声処理は不要です。

ソフトウェア設定項目選択デバイス
語橋マイク入力実際のマイク
語橋翻訳出力仮想デバイス A Input
語橋私の翻訳再生オフ
OBS/会議ソフトマイク入力仮想デバイス A Output

よくある間違い​

❌ 間違い 1：両チャンネルで同じ仮想デバイスを使用 → フィードバックループ

私の翻訳出力と相手の音声入力が同じ仮想デバイスを使用すると、翻訳音声が繰り返し収集され、エコーが爆発します。
✅ 解決策：2 つのチャンネルは必ず異なる仮想オーディオデバイスを使用してください。

❌ 間違い 2：会議ソフトのマイクを実際のマイクに設定 → 原文を送信

相手は未翻訳の原文を聞くことになり、同時通訳の意味がなくなります。
✅ 解決策：会議ソフトのマイクは必ず仮想デバイス A Output に設定してください。

❌ 間違い 3：相手の翻訳出力を仮想デバイスに設定 → 会議にクロストーク

相手の翻訳音声が会議ソフトに収集されて送信され、クロストークが発生します。
✅ 解決策：相手の翻訳出力は必ず実際のヘッドフォンに設定し、ローカル再生のみにしてください。
